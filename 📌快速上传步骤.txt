📌 快速上传到 GitHub 的3个步骤
===============================================

第一步：运行上传脚本
--------------------------
1. 找到项目文件夹中的 "上传到GitHub.bat" 文件
2. 双击运行它
3. 等待脚本自动完成所有操作
4. 看到 "✅ 上传完成！" 表示成功


第二步：启用 GitHub Pages
--------------------------
1. 在浏览器打开：https://github.com/xun6666/guihuanong-h5
2. 点击仓库顶部的 "Settings"（设置）
3. 在左侧菜单找到 "Pages"
4. 在 "Source" 下拉菜单中：
   - Branch: 选择 "main"
   - 文件夹: 选择 "/ (root)"
5. 点击 "Save" 按钮
6. 页面会显示：Your site is ready to be published at...


第三步：访问您的网站
--------------------------
等待 1-2 分钟后，访问：

https://xun6666.github.io/guihuanong-h5/

✅ 这就是您的H5页面链接！


生成二维码
--------------------------
1. 复制上面的链接
2. 双击打开 "二维码生成器.html"
3. 粘贴链接
4. 点击 "生成二维码"
5. 下载保存


===============================================
⚠️ 如果遇到问题
===============================================

问题1：运行脚本时提示 "git不是内部或外部命令"
解决：需要先安装 Git
1. 访问 https://git-scm.com/download/win
2. 下载并安装 Git for Windows
3. 安装完成后重新运行脚本


问题2：推送时要求输入用户名和密码
解决：
- 用户名：输入您的 GitHub 用户名 (xun6666)
- 密码：需要使用 Personal Access Token (不是GitHub密码)
  
  生成 Token 的方法：
  1. GitHub 右上角头像 → Settings
  2. 左侧最下方 Developer settings
  3. Personal access tokens → Tokens (classic)
  4. Generate new token
  5. 勾选 "repo" 权限
  6. 生成后复制 Token，粘贴作为密码使用


问题3：推送被拒绝 (rejected)
解决：可能是仓库已有内容
运行：git pull origin main --allow-unrelated-histories
然后再运行：git push -u origin main


问题4：GitHub Pages 404 错误
解决：
1. 检查 Settings → Pages 是否正确配置
2. 确认选择的是 main 分支
3. 等待几分钟，GitHub 需要时间部署
4. 清除浏览器缓存重试
